A beginners guide to CSS layout

[ Part 1: Display & Positioning ]

SUBHEADER

Website styling and layout have come a long way since web designers were restricted to the limited design options provided by HTML alone. Cascading Style Sheets or CSS have revolutionised website design by creating a degree of separation between design and content, resulting in greater control over styling, ease of use, consistency and smaller file sizes.

“CSS is all about changing and improving the appearance of HTML
(McFarland, 2009, p. 01).”

Understanding the basic properties of CSS is fundamental to web design today; in particular the ability to recognise how styles can manipulate HTML page elements to create solid, cross-browser compatible layouts is invaluable.

This guide has been created to assist beginner web designers’ in comprehending the basic CSS display and positioning properties by providing a visual preview of how their values behave. As a result visual learners are able to gain a greater understanding of the concepts.


Keywords: CSS, display, positioning, layout, basic, beginner, visual, preview, demonstration, tutorial, guide.

Display

The display property determines how a HTML element is displayed by controlling the type of rendering box used by the element. Each HTML element is assigned A default display property, however the default property can be overridden by defining an alternative property (Hunt, 2010). The three main display properties are:
display:block;
display:inline;
display:none;

Many other display properties are also available such as inline-block, list-style, table & table-cell.

Position

The position property allows you to position elements anywhere the screen. The values for the position property are: position:static; – This is default.
position:relative
position:absolute


The top or bottom & left or right properties can be used in conjunction with the position: relative; and position:absolute; properties to specify the location of the element.

Static
  • Static is the default positioning for all elements.
  • It follows the standard flow of the document.
  • It can NOT be moved from it’s natural position with the top or bottom, and left or right properties.
Relative
  • Similar to static in that it follows the standard flow of the document.
  • However it also allows you to move or rather project the element from its initial standard position, leaving the initial flow of the document unchanged with a space left where the element was originally positioned.
  • The properties top or bottom, and left or right will move the element in relation to its original position.
Absolute
  • Unlike static and relative, absolute pulls the element out of the standard flow of the document giving you complete control so you can place it exactly where you want it to go.
  • Because the element has been pulled out of the standard flow (unlike relative which stays in the flow) the elements around it will be re-repositioned to fill in the space it once occupied.
  • The values: top or bottom, and left or right will move the element in relation to its containing block.
Nesting elements - relative & absolute
  • If an element is defined as relative all the elements nested within it will be positioned in relation to it. For example an absolute element that is nested within an element assigned the property position:relative; it will move in relation to the edges of that containing element.
  • However if an absolute element is not nested within an element assigned the property position:relative; it will move in relation to the edges of the page (browser window).
Fixed
  • Similar to absolute a fixed element will move out of the standard flow, however a fixed element will remain fixed in the window and not scroll with the rest of the document.This can be handy if you want something to always be in view no matter how much content there is.
  • However unlike absolute positioned elements, fixed elements cannot be contained by relative parent elements.
  • Note: position:fixed; is not supported by IE6.

CSS Critical Analysis

Although CSS provides a vast improvement on HTML styling, there is room for improvement. CSS was ahead of its time when it was first introduced, freeing designers from the standard table layout; however as the web has evolved CSS has fallen behind.


“Conceived in an age when the design of most web sites still looked quite plain, CSS creators couldn’t anticipate the richness and intricacy of the designs that it would eventually be asked to describe (Yank, 2008).”


Although experienced web designers have developed techniques to manipulate CSS into achieving outcomes never envisioned by its creators, novice designers can encounter difficulties mastering these methods, and workarounds can be problematical and time-consuming.


Vertical alignment is a classic example where CSS has struggled to catch up, and a seemingly simple request is not so simple. Kistner (2004) explains this in more detail (http://phrogz.net/css/vertical-align/index.html), highlighting the fact that the vertical-align property only works as expected in tables, with no simple property or value provided for normal block elements, such as the margin: auto value which makes horizontal alignment effortless.


Collapsing vertical margins also provides a great level of confusion to beginners, particularly in the context of nested elements. Olsson et al. (2008) describes the occurrence, as when the vertical margins of two elements are touching, only the element with the largest margin will be used. A margin should behave as expected in any circumstance to separate all elements with no further style specifications needed, this is not the case with CSS.


Other properties, such as, nested floats also come with some interesting behaviours. The main one being that floated items are not recognised by their parent container causing it to collapse, which in turn leads to other messy issues. CSS layouts would be greatly simplified if elements consistently conformed to their display behaviour guidelines.

Questions & Answers

  1. Why is it important to understand the fundamental rules of CSS when trying to layout a page?
    CSS has many different properties that can react differently in certain situations. Understanding the fundamental rules will assist in identifying which methods will provide a more structurally sound and valid website, which in turn will create consistency when the site is viewed in multiple browsers.
  2. What display value is most commonly used when building the structure of a website, and why?
    The display block value is used, as block elements, such as div’s can be resized and also nest other elements within them.
  3. When would visibility:hidden; be more appropriate than display:none;?
    When you want the original space occupied by the element kept in tact, but the element to be hidden from view.
  4. Why is it undesirable to use a large amount of absolute positioning on your web page?
    Because absolute positioning takes elements out of the natural flow of the document, they don’t react to all of the elements on the page, which can cause problems to arise if they are used in excess and the browser window is resized.
  5. If you don’t define the top or bottom, and left or right properties where will an absolute element position itself?
    If there are no static or relative elements surrounding it, it will jump up to the top, left corner of the browser window.
